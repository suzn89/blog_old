BEM 방법론

2019-03-17

CSS 방법론 (Methodologies)

CSS 방법론은 쉽게 유지보수가 가능하게 하고, 재사용 할 수 있도록 구현하며, 쉽게 확장가능하게 하는 공통의 목적을 가진다.

BEM (Block Element Modifier)

작명규칙 (Naming Convention)

선택자의 이름은 명확하게
소문자,숫자만을 이용하여 작성

Block

블럭 요소들을 정의하고, 이 큰 덩어리(블록)들의 클래스 어근을 맨 앞에 붙여 그 블럭이 포함하는 요소들의 클래스명을 짓기 시작하면 된다.

  • 블록은 class 속성으로 표시
  • 태그,id 선택자를 사용하면 안 됨
  • 블록은 서로 중첩해서 사용이 가능
  • 형태가 아닌 목적에 맞게 결정할 것

Element

요소(element)는 블럭을 구성하는 조각이다.
이 요소들은 두개의 언더바(underscore)로 연결하여 블럭 다음에 위치시킨다.
두개의 언더바로 연결하는 것은 시각적으로 쉽고 빠르게 코드를 찾고 조작 할수 있도록 돕는다.

  • 형태가 아닌 목적에 맞게 결정할 것
  • 요소는 중첩해서 사용이 가능
  • 블록의 부분으로서만 사용이 가능하고, 다른 요소의 부분으로는 사용이 불가
  • 블록안에 요소가 없을 수도 있음

Modifiers

수식어(modifier)는 블럭이나 요소의 속성을 나타낸다.
이 속성은 블럭이나 요소의 외관이나 상태를 변화 시키는 것을 나타내는데, 요소의 스타일이 같은 것이라면 사이트의 다른 영역이어도 새로운 클래스를 정의하지 않아도 괜찮다.
사용할때는 블럭또는 요소의 다음에 두개의 하이픈을 추가하여 modifire를 사용한다.

  • 블록이나 요소의 모양,상태를 정의
  • 블리언 타입과 키-벨류 타입이 있음
  • 블리언 타입 - 수식어가 있으면 값이 true라고 가정(form__btn--disabled)
  • 키-벨류 타입 - 키,벨류를 하이픈으로 연결하여 표시(color-red)
  • 수식어(modifier)는 단독 사용이 불가

예를 들자면,

  • HTML 마크업을 모듈화 하여 사용이 가능
  • 블럭요소들을 믹스인이 가능한 모듈이라는 개념
  • .block(블록요소)__element(내부요소)–modifier(기능)
  • 직관적인 클래스명
  • 전체 DOM tree 구조를 보지 않아도 됨
  • 때문에 유지보수가 용이함
  • 기존 중복된 클래스 사용시 레이아웃이 변경되면 만나게되는 스타일시트의 대량 수정을 줄일 수 있다.
  • 쿼리를 단순화 하기 위해 길고 유니크한 클래스명 사용
